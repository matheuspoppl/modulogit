RESUMO DAS AULAS DE GIT - VERSIONAMENTO PROCESSO --> CRIA/ALTERA ARQUIVO(S) / ADICIONA NO BRANCH / COMITA NO BRANCH / EXPORTA PRO GIT - cd C:\xampp/htdocs/(nome da pasta do projeto) - entra pelo GitBash na pasta onde está salvo o projeto e onde estão os arquivos que serão inclusos no branch. depois que entrar na pasta dar um git init para iniciar o git no branch que quer usar, no branch atual. - git log - resulta no historico de alterações do repositorio - git status - resulta na situação atual de alterações dos arquivos do repositório - git init - para iniciar o git - git add -A - adiciona tudo o que foi alterado ao branch (necessário antes de comitar) - git commit -am "comentário" - ANTES DE COMITAR USE UM GIT STATUS PARA VERIFICAR SE TUDO ESTÁ ADICIONADO E PRONTO PARA SER COMITADO - comita e atualiza as alterações feitas no arquivos do branch - git reset --hard (hash do commit) - volta o repositório para a situação do commit(hash) usada deletando tudo o que aconteceu no repositório depois daquele momento, volta pra situação daquele momento (hash usada) - git reset --soft (hash do commit) - volta o repositório para a situação do commit(hash) usada mantendo as alterações dos arquivos, porém sem comitá-las, volta pra situação daquele momento (hash usada) antes de ter sido salvo - git branch - resulta na lista de todos os branchs que existem e o que tiver com * é o que está sendo usado no momento, é o atual que eu estou - git branch (nome do branch) - cria um novo branch com o nome citado importando todos os arquivos e alterações feitas do branch a partir do qual o novo branch foi gerado, ou seja, cria uma cópia do branch atual com o nome citado. - git checkout (nome do branch) - troca de branch para o branch citado - git diff - mostra todas as alterações detalhadamente, linha por linha, feitas em todos os arquivos do branch atual - git diff --name-only - resulta apenas os nomes dos arquivos em que houveram alterações - git diff (nome do arquivo.extensão) - resulta todas as alterações feitas apenas no arquivo citado apenas - git checkout HEAD -- (nome do arquivo.extensão) - volta todas as alterações feitas no arquivo que ainda não foram comitadas dentro do branch atual, o HEAD significa que é para ser feito no branch atual esse comando. Volta para o estado "original" do arquivo. - git remote add origin https://github.com/matheuspoppl/modulogit.git - adiciona/conecta um repositório remoto chamado origin com o endereço citado ao branch atual. - git remote - resulta na lista de repositórios adicionados - git remote -v - traz com mais detalhes do repositório - git push -u origin master - exporta o branch master para o repositório origin (remoto) - git push origin master - atualiza o repositório origin com todo o conteúdo NOVO do branch master - arquivo .gitignore - arquivo no qual estão os nomes dos arquivos, pastas, diretórios que não vão aparecer no branch, seja no repositório online, seja no local - ex: index.html (vai ignorar apenas o arquivos), *.txt (vai ignorar todos os arquivos .txt do branch), assets/ (vai ignorar todo o conteúdo da pasta assets)
