RESUMO DAS AULAS DE GIT - VERSIONAMENTO Caso tenha necessidade de linkar um pc novo, gerar uma nova chave SSH (https://help.github.com/articles/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent/) PROCESSO --> CRIA/ALTERA ARQUIVO(S) / ADICIONA NO BRANCH / COMITA NO BRANCH / EXPORTA PRO GIT - cd C:\xampp/htdocs/(nome da pasta do projeto) - entra pelo GitBash na pasta onde está salvo o projeto e onde estão os arquivos que serão inclusos no branch. depois que entrar na pasta dar um git init para iniciar o git no branch que quer usar, no branch atual. - git log - resulta no historico de alterações do repositorio - git status - resulta na situação atual de alterações dos arquivos do repositório - git init - para iniciar o git - git add -A - adiciona tudo o que foi alterado ao branch (necessário antes de comitar) - git commit -am "comentário" - ANTES DE COMITAR USE UM GIT STATUS PARA VERIFICAR SE TUDO ESTÁ ADICIONADO E PRONTO PARA SER COMITADO - comita e atualiza as alterações feitas no arquivos do branch - git reset --hard (hash do commit) - volta o repositório para a situação do commit(hash) usada deletando tudo o que aconteceu no repositório depois daquele momento, volta pra situação daquele momento (hash usada) - git reset --soft (hash do commit) - volta o repositório para a situação do commit(hash) usada mantendo as alterações dos arquivos, porém sem comitá-las, volta pra situação daquele momento (hash usada) antes de ter sido salvo - git branch - resulta na lista de todos os branchs que existem e o que tiver com * é o que está sendo usado no momento, é o atual que eu estou - git branch (nome do branch) - cria um novo branch com o nome citado importando todos os arquivos e alterações feitas do branch a partir do qual o novo branch foi gerado, ou seja, cria uma cópia do branch atual com o nome citado. - git checkout (nome do branch) - troca de branch para o branch citado - git diff - mostra todas as alterações detalhadamente, linha por linha, feitas em todos os arquivos do branch atual - git diff --name-only - resulta apenas os nomes dos arquivos em que houveram alterações - git diff (nome do arquivo.extensão) - resulta todas as alterações feitas apenas no arquivo citado apenas - git checkout HEAD -- (nome do arquivo.extensão) - volta todas as alterações feitas no arquivo que ainda não foram comitadas dentro do branch atual, o HEAD significa que é para ser feito no branch atual esse comando. Volta para o estado "original" do arquivo. - git remote add origin https://github.com/matheuspoppl/modulogit.git - adiciona/conecta um repositório remoto chamado origin com o endereço citado ao branch atual. - git remote - resulta na lista de repositórios adicionados - git remote -v - traz com mais detalhes do repositório - git push -u origin master - exporta o branch master para o repositório origin (remoto) - git push origin master - atualiza o repositório origin com todo o conteúdo NOVO do branch master - arquivo .gitignore - arquivo no qual estão os nomes dos arquivos, pastas, diretórios que não vão aparecer no branch, seja no repositório online, seja no local - ex: index.html (vai ignorar apenas o arquivos), *.txt (vai ignorar todos os arquivos .txt do branch), assets/ (vai ignorar todo o conteúdo da pasta assets) - git revert --no-edit (hash do commit a ser revertido) - reverte o commit feito commitando, ou seja, desfaz tudo o que foi feito do commit (citado na hash) mantendo o registro/histórico de alterações feitas nos arquivos. A diferença entre o revert e o reset é que: o revert reverte as alterações feitas no arquivos deixando registrado no log e o reset volta o arquivo ao estado que estava no momento do commit. - git push origin (nome_branch_aSerAdicionado) - sobe para o servidor remoto o branch com o nome citado depois do origin, ou seja, cria um novo branch no servidor remoto. - git push origin :(nome_branch_aSerAdicionado) - deleta do servidor remoto o branch citado, os dois pontos ":" significam que é pra deletar o arquivo. - git branch -D (nome_branch) - deleta do repositorio local o branch citado. - git pull origin master - comando que atualiza o repositório local de acordo com repositório remoto, ou seja, traz todos os arquivos do origin (remoto) para o master (local) - deve ser feito sempre antes de começar a editar os arquivos no repositorio local para deixar ele alinhado com o remoto. - git clone (url_doProjeto) - clona um repositório remoto que está sendo citado na url, é necessário estar na pasta na qual o repositório será salvo, ou seja, pelo GitBash (cd C:\xampp/htdocs/nomePastaNova) vai até a pasta na qual estará o clone e aí executa o comando.
